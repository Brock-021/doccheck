# DocCheck · 使用手册

**DocCheck** 是一款基于 AI 的文档合规检查工具。用户上传 Word 文档（.docx），系统自动调用大语言模型（LLM）进行规则检查，生成检查报告，支持审核与流程闭环。

## 3. 文档编写者操作

### 3.1 上传文档检查

1. 登录系统，进入 **上传检查** 页面
2. 选择 **文档类型**（管理制度 / 技术方案 / 操作手册）
3. 选择 **检查阶段**（初稿 / 终稿）
4. 拖拽或点击选择 .docx 文档
5. 点击 **开始检查**
6. 系统自动跳转到检查进度页，完成后可查看报告

> **提示**：检查过程通常 5~30 秒，取决于文档大小和规则数量。

### 3.2 查看检查报告

在 **工作台** 或 **文档列表** 中找到已检查的文档，点击 **查看报告**。

报告包含：

- **摘要**：总规则数、通过数、未通过数、通过率
- **逐条结果**：每条规则的检查结论
  - ✅ **通过**：文档符合该规则
  - ❌ **不通过**：文档存在问题，有具体问题描述和修改建议
  - ❓ **无法判断**：LLM 无法确认，需人工审核

### 3.3 查看审核结论

审核员审核后，编写者可在报告中查看审核结果：

- **确认**：问题已确认，等待修改
- **驳回**：指出的问题不成立，驳回需写备注说明原因
- **忽略**：该问题可接受，不必修改，需写备注说明原因
- **结论**：审核员给出整体结论（通过/需修改/不通过）

### 3.4 查看历史版本对比

文档多次上传检查后，可进入 **历史对比** 页面，并排查看不同版本检查结果的变化（新增问题、已解决问题、持续问题等）。

## 4. 审核人员操作

### 4.1 审核检查结果

1. 进入 **工作台**，查看待审核的报告列表
2. 点击报告，进入审核页面
3. 逐条处理每条检查结果：

| 操作 | 含义 | 备注要求 |
|:----|:-----|:---------|
| **确认** | 确认该问题存在，需修改 | 可选（自动生成确认信息） |
| **驳回** | 认为该问题不成立 | 必填（说明驳回理由） |
| **忽略** | 问题存在但可接受，不改 | 必填（说明忽略原因） |

4. 全部处理完后，提交 **审核结论**：

| 结论 | 含义 |
|:----|:-----|
| **通过** | 文档合规，无需修改 |
| **需修改** | 存在问题，文档需修改后重新检查 |
| **不通过** | 文档严重不合格 |

### 4.2 导出报告

在报告详情页或审核页面，点击 **导出** 按钮：

- **PDF**：适合分享给外部或打印
- **Word**：适合内部流转

## 5. 管理员操作

### 5.1 规则管理

**管理 → 规则管理**

管理员可以：

- **查看规则列表**：按文档类型筛选
- **新增规则**：填写规则名称、描述、严重程度、所属文档类型
- **编辑规则**：修改规则内容
- **启停规则**：临时启用/禁用某条规则
- **删除规则**：移除不需要的规则

每条规则包含：

| 字段 | 说明 | 示例 |
|:----|:-----|:-----|
| 名称 | 规则简短名称 | 格式规范 |
| 描述 | 使用自然语言描述规则 | 文档标题应使用一级标题（Heading 1） |
| 严重程度 | must_fix（必须改）/ suggest（建议改） | must_fix |
| 阶段 | initial（初稿）/ final（终稿）/ all（全部） | all |
| 所属类型 | 适用于哪种文档类型 | 管理制度 |

### 5.2 文档类型管理

管理员可以新增、编辑、删除文档类型。每种类型可绑定不同的检查规则集合。

当前预置的文档类型：

- **管理制度**：公司规章制度类文档
- **技术方案**：技术设计/实施方案
- **操作手册**：操作指南/用户手册

### 5.3 用户管理

**管理 → 用户管理**

管理员可以：

- **查看用户列表**：所有用户及状态
- **新增用户**：创建新账号
- **编辑用户**：修改显示名、角色
- **禁用用户**：临时停用账号
- **启用用户**：恢复账号
- **重置密码**：生成新密码

### 5.4 LLM 配置

**管理 → LLM 配置**

配置 AI 检查引擎的模型参数（见部署手册第 4 节）。

### 5.5 审计日志

**管理 → 审计日志**

记录所有关键操作，包括：

- 用户登录/登出
- 文档上传
- 规则修改
- 用户管理操作
- LLM 配置变更

支持按操作类型和时间范围筛选。

## 6. 常见场景

### 场景一：编写管理制度文档

```
1. 管理员创建"管理制度"类型的检查规则
2. 编写者登录 → 上传检查 → 选择"管理制度" → 上传 .docx
3. 系统自动检查 → 生成报告
4. 编写者查看报告，根据问题逐条修改
5. 重新上传 → 再次检查
6. 审核员审核报告 → 给出结论"通过"
7. 编写者查看最终结论 → 文档定稿
```

### 场景二：批量检查多个文档

每次上传一个文档，系统会自动检查。可以在工作台查看所有文档的检查状态。

### 场景三：报告归档

检查完成后，可导出 PDF 或 Word 报告，与文档一并归档。

## 7. FAQ

### Q: 支持哪些文档格式？
目前仅支持 `.docx` 格式（Word 文档）。单个文件最大 50MB。

### Q: 检查规则支持哪些内容？
规则使用自然语言描述，LLM 会自动理解并判断。例如：
- "文档标题应使用一级标题"
- "正文应使用全角中文标点"
- "技术方案应包含系统架构描述"

### Q: 检查速度如何？
通常 5~30 秒完成一次检查，取决于文档长度和规则数量。

### Q: 检查结果准确吗？
LLM 检查为辅助工具，建议所有检查结果由审核员人工复核后再做最终结论。

### Q: 可以自定义检查规则吗？
可以。管理员在规则管理页面，用自然语言新增规则即可，无需技术背景。

### Q: 如何查看谁上传了文档？
文档列表页面会显示上传者和上传时间。

### Q: 文档会保存多久？
文档和检查记录永久保存（除非管理员手动清理数据库或上传目录）。

### Q: 检查时断网了怎么办？
检查过程由后台异步执行，即使页面关闭，检查也会继续。重新打开报告页面即可查看结果。

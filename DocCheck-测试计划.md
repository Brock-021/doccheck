## 一、测试目标

1. 验证规则管理（增删改查、启用禁用、复制、排序）功能正确
2. 验证文档上传与 AI 检查全链路正常
3. 验证检查报告展示与审核流程正确
4. 验证用户认证与角色权限控制有效
5. 验证系统管理功能（LLM 配置、用户管理、审计日志）可用
6. 验证非功能需求（性能、安全、可用性）达标

## 三、测试策略

### 3.1 测试层次

```
┌──────────────────────────────┐
│        E2E 端到端测试          │ 模拟用户真实操作流程
├──────────────────────────────┤
│     集成测试（API 接口）       │ 验证接口间的交互
├──────────────────────────────┤
│      单元测试（服务层）        │ 验证核心逻辑
├──────────────────────────────┤
│      静态分析（Lint/Type）     │ 代码质量门禁
└──────────────────────────────┘
```

### 3.2 测试类型

| 类型 | 覆盖 | 执行方式 |
|:----|:-----|:---------|
| **单元测试** | services/ 下各模块核心逻辑 | pytest |
| **API 集成测试** | routers/ 下各接口 | pytest + httpx TestClient |
| **UI 功能测试** | 前端模板渲染、表单交互 | pytest + playwright（可选）|
| **安全测试** | 未登录访问、越权操作、文件上传安全 | 手动 + 脚本 |
| **性能测试** | 并发、响应时间、文档检查耗时 | locust 或 pytest-benchmark |
| **兼容性测试** | 不同浏览器、不同 .docx 格式 | 手动 |

### 3.3 测试环境

| 环境 | 用途 | 配置 |
|:----|:-----|:------|
| **开发环境** | 开发自测 | 开发机 |
| **测试环境** | 功能验证、集成测试 | 2核2G 测试服务器 |
| **预发布环境** | 验收测试 | 同测试环境 |
| **生产环境** | — | 2核2G 正式服务器 |

### 3.4 测试数据

| 数据 | 说明 |
|:----|:------|
| **测试用户** | admin(系统管理员), reviewer(审核), writer(编写者), multi(多角色) |
| **测试文档类型** | 立项报告、技术规格说明书、需求文档、验收报告 |
| **测试规则** | 每种文档类型 5~8 条规则，含必须改/建议改 |
| **测试文档** | 包含规范的 demo 文档 + 有缺陷的 demo 文档 + 空文档 + 大文档(>10MB) |
| **恶意文件** | 非 .docx 文件、带宏的文件、超大文件 |

## 六、通过标准

### 6.1 每轮测试进入标准

- 本轮所有功能代码已提交
- 单元测试通过率 100%
- 本轮变更相关的 API 测试通过率 100%

### 6.2 测试通过标准

| 指标 | 要求 |
|:----|:------|
| 单元测试通过率 | 100% |
| API 测试通过率 | 100% |
| 功能测试通过率 | ≥95%（P0/P1 级别缺陷不允许） |
| P0 级别缺陷 | 0 个 |
| P1 级别缺陷 | ≤3 个 |
| 性能测试 | 单文档检查 ≤60s，页面加载 ≤2s |

### 6.3 缺陷等级定义

| 等级 | 定义 | 处理要求 |
|:----|:-----|:---------|
| **P0 致命** | 系统崩溃、核心功能（上传/检查/审核）不可用、数据丢失 | 立即修复，阻塞发布 |
| **P1 严重** | 功能异常但可替代方案、数据错误但不丢失 | 本轮发布前修复 |
| **P2 一般** | 非核心功能异常、UI 错位、文案错误 | 可延后到下一版本 |
| **P3 建议** | 体验优化、样式微调 | 可记录做后续优化 |

## 七、风险与应对

| 风险 | 缓解措施 |
|:----|:---------|
| LLM API 不可用导致测试阻塞 | 使用 Mock LLM 服务替代真实调用 |
| 测试文档格式不全面 | 准备多种格式的 .docx（有章节/无章节、带表格/带图片）|
| 2核2G 性能测试结果不理想 | 明确性能基线，识别瓶颈（通常是 LLM 调用耗时）|
| 审核流程场景遗漏 | 用状态转换矩阵覆盖所有组合 |
